ARCHITECTURE OVERVIEW
================================================================================

TECHNOLOGY STACK:
  Frontend: Next.js 15 (App Router) + TypeScript + Tailwind
  State: Zustand (single store)
  Wallet: @solana/wallet-adapter-react + @solana/wallet-adapter-react-ui
  HTTP: Axios API client
  RPC: @solana/web3.js (Connection)
  Forms: react-hook-form
  Testing: Jest + @testing-library

DATA FLOW:
  User → Wallet Adapter → Zustand Store → Components
                    ↓
                API Client → Backend (FastAPI)
                    ↓
                Returns VaultState → Store → UI Update

REAL-TIME UPDATES:
  Phase 1: Polling (5s for vault, 30s for balance) ✓ MVP
  Phase 2: WebSocket subscription (future enhancement)

API CONTRACTS
================================================================================

EXISTING ENDPOINT:
  GET /vault/{vault_id}
  Response: VaultState { balance, cumulative_pnl, num_trades, ... }

NEEDS IMPLEMENTATION (Backend):
  POST /vault/{vault_id}/create
  POST Body: { authority, initial_balance, position_limit_bps, max_drawdown_bps }
  
  GET /vaults?authority={wallet_address}
  Response: VaultState[]

================================================================================
CUSTOM HOOKS
================================================================================

1. useWalletBalance(publicKey)
   → Returns: { balance, loading, error, refetch }
   → Behavior: Polls chain every 30s, updates store

2. useUserVaults(authority)
   → Returns: { loading, error, refetch }
   → Behavior: Fetches user's vaults from API on authority change

3. useVaultPolling(vaultId, interval?)
   → Returns: { loading, error, refetch }
   → Behavior: Polls vault state every 5s, updates store, cleanup on unmount

================================================================================
ERROR HANDLING
================================================================================

Wallet Errors:
  • Wallet not installed → Show install link
  • Connection rejected → Show "User rejected" message
  • Network mismatch → Show "Switch network" message

API Errors:
  • Vault not found (404) → Return to vault list
  • Create vault fails → Keep form populated, show retry
  • Balance fetch fails → Use cached balance, show warning

Form Validation:
  • Balance ≤ 0 → "Must be greater than 0"
  • Position limit invalid → "Must be 0-100%"
  • Max drawdown invalid → "Must be 0-100%"

User Feedback:
  • Success: Toast notification
  • Error: Alert with error message
  • Loading: Spinner + loading text
  • Retry: Error alert with retry button
